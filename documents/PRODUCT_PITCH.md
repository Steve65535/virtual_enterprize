# OpenClaw Enterprise (Virtual Enterprise) 产品说明书

OpenClaw Enterprise 是一款基于 Tauri (Rust) + React 构建的**本地化私有 AI 编排引擎与隔离运行环境**。本产品旨在将多智能体 (Multi-Agent) 的调度、工作流与实际物理计算环境进行深度绑定，为企业或个人提供开箱即用、数据绝对安全、互联互通的数字员工（Agent）运行集群。

系统设计强调**“算力本地化、数据隔离化、沟通自动化”**，让每一个 AI Agent 都在受宿主机绝对控制的独立 Docker 沙箱中进行计算、开发和协作，无需依赖任何不受控的第三方云端执行环境。

---

## 一、 核心功能特性 (Features)

### 1. 私有化沙箱隔离环境 (Local Docker Sandbox)
- **按需拉起计算节点**：系统一键创建搭载 Python、Git 等极客基础组件的独立 Docker 环境（容器），每个“数字员工”均有独立的运行空间。
- **资源精细化限流**：可对不同角色的 AI 节点独立限制最大内存使用量（256MB/512MB/1GB等）和 CPU 核心数，保障宿主机的稳定性与资源合理调配。
- **无感秒级启动与销毁**：底层 Rust 通过系统级线程与本地 Docker 引擎进行异步交互，拉起/销毁数字员工环境无需阻塞用户界面，且随用随抛。

### 2. 动态参数与凭据注入架构 (Config Injection)
- **模板零感知拷贝**：容器启动前，系统会将清洁后的 `openclaw-base` 母盘文件模板拷贝至为其分配的本地存储卷重。
- **密钥热挂载策略**：在前端设置的全局 LLM API Keys（如 DeepSeek/OpenAI）、以及具体的业务沟通渠道参数（Feishu / Lark / Discord AppID与Secret），会在容器启动的瞬间由 Rust 代码自动覆写到该容器的 `openclaw.json` / `models.json` 配置中，并封装进 `.openclaw.env`。
- **凭据不落地**：开发者的真实密钥和 Token 等无需经过代码硬编码或手动挂载外部系统，确保了数据传输在流转侧的极端安全。

### 3. 数据壁垒与双网关控制 (Network Security & Topology)
系统原生搭载并自动编排两套子网逻辑，解决 Agent 极易造成的安全泄露问题：
- **`openclaw-intranet` (强制局域网)**：所有启动的 AI 节点均被强行拉入该纯净隔离子网，相互之间依靠基于名字的 DNS Alias 直接相互连通 (例如基于 HTTP RPC 或 socket 通信)。
- **`openclaw-internet` (断网控制阀)**：系统支持通过 UI 面板进行热插拔的颗粒度网络管控。通过断开此互联网网桥，可以彻底锁死指定 AI 节点的公网通讯权限。处于该状态下的 AI 仅能阅读内部文件或同僚传导的数据并发回本地结果，物理上截断了数据向公网外发、或被反向注入的风险。

### 4. 跨沙箱物理协作机制 (Inter-Agent Collaboration)
- **共享物理总线磁盘 (`/enterprise_shared`)**：为了打破各 AI 数字员工的数据壁垒，系统在 macOS 宿主机级创建持久化目录，并平行挂载到所有微服务沙箱内。多路 Agent 可对该目录以零开销的方式进行文件读写交换（例如 A爬虫程序存入 10GB 结构化数据，B数据分析节点随后直接读取）。
- **极轻量底层调度总线 (Local.Bus)**：废弃了易断连的全局 WebSocket。所有沙箱将特定的互传信件写入到 `/enterprise_shared/.bus`，由核心引擎的 `notify-rust` 钩子以毫秒级触发响应并拦截、分发，实现了 Agent 间的脱飞书本地高效短链沟通。

### 5. 可视化中央控制台 (React Dashboard)
- 提供全面的资源看板，实时监测当前宿主机的在跑示例数、分别占用的 CPU 与内存情况。
- **间谍雷达 `BusMonitor`**：直观打印正在发生的沙箱底层的每一次 `Local.Bus` 跨节点报文互传记录及网络流转状态。
- **工作台切换**：直接提供挂载在该实例的 `ChatTerminal` (用于接收 stdout 与直调 shell) 以及 `FileTree` (基于系统事件监听挂载卷的动态文件树查看器)。

## 二、 实际落地场景推演 (Use Cases)

- **场景 A：本地财务与数据隔离化审查**
  创建不可联网的 Agent，为其注入离线运行的模型。人类用户在飞书中请求其进行数据分析，Agent 读取本地共享挂载磁盘中的机密财报报表，生成分析图像后通过管道投递回企业共享盘。全流程无数据上行公网。
- **场景 B：复杂链路开发与自治微服务群**
  创建一个拥有完全公网权限的 Dev Agent，以及一个断网的 QA Agent。Dev 编写最新爬虫逻辑并访问外部网站落盘，QA 系统检测到文件树异动自动启动本地离线测试，最后在 `.bus` 总线留下测试结论发回前端控制台，由人类老板最终检阅。
- **场景 C：多模型协同 (Ensemble Models)**
  建立多个不同大模型 API (DeepSeek 处理逻辑，OpenAI 处理翻译) 驱动的功能沙箱，通过内置内网互相分步骤投递数据与需求，实现在局域网环境下复杂任务的一条龙自动化流水线处理。
