# Phase 5: 本地数字协作网络 (当前进度与产品边界)

*生成时间：2026年3月11日*

截至目前，本项目关于 **Phase 5 (内网互联协作)** 的底层引擎能力已由前端/Rust 侧开发全面攻克。同时我们明确了本系统“前后端分离、内网协作但外网触达”的最终产品边界。

## 1. 核心架构已完成项 (Rust 底座)

### 1.1 物理共享挂载卷 ($ enterprise_shared)
- **实现细节**: 底层已强制创建全局宿主机目录 `/tmp/openclaw_enterprise_shared`，并在每次 `start_sandbox` 拉起容器时，通过 `-v` 参数将其挂载至每一个员工环境的 `/enterprise_shared` 中。
- **效果**: 所有独立的沙箱，现在可以秒级通过这个全局共享的物理磁盘互传大体积数据或文件。

### 1.2 双网关物理隔离 (Intranet & Internet)
- **实现细节**: 摒弃了默认桥接网，改用定制网络。Rust 在应用初始化时会预建两个网络：
  - `openclaw-intranet` (内部隔离网，带 `--internal` 标识)
  - `openclaw-internet` (外连网)
- **效果**: 
  - 所有新入职员工，强制连入 `openclaw-intranet` 并且带有以自己名字命名的 DNS Alias (例如：Employee A 可以直接 `ping 员工B`)。
  - Dashboard 界面支持对单个员工拔除 `openclaw-internet` 外网连接。不仅做到了员工间微服务级的高效局域网通信，更做到了金融级的数据防泄漏。

### 1.3 极简轻量级 Local IPC 消息总线 (.bus)
- **实现细节**: 在共享盘下方，创建了一个隐藏目录 `/tmp/openclaw_enterprise_shared/.bus`。使用类似于 UNIX domain sockets 的核心原理。
- **效果**: 取代了庞大臃肿的系统级 WebSocket，员工通过往该隐藏目录写特定格式的数据，触发系统的原生文件监视器 `notify_rust`，由此实现数字员工向老板大盘发出毫秒级的本地消息提醒。

## 2. 最终产品边界确立 (C端输出层口)

此前曾讨论是否要抛弃“飞书”等 IM 软件完全转投本地界面沟通。经评估明确：
**在企业级产品的最终交互链路上，飞书/微信等办公 IM 软件不可替代。**

- **系统定位**: OpenClaw Enterprise 相当于一家公司的**隐秘数据中心与黑盒办公室**。它的界面主要服务于“公司老板 (Admin)”进行资源调配、员工断网、拉起微服务。
- **业务终端**: 普通人类用户（C端操作者）不需要面对复杂的 `ls` 面板与文件树。他们应该继续使用飞书在群内向员工发号施令。
- **业务闭环**:
  1. 人类在飞书群 @经理。
  2. 黑盒中的经理收到飞书回调，在内网 (`openclaw-intranet`) 中分发任务给内部的爬虫和数据分析员工。
  3. 各模块员工在本地隔离环境协同完成，将报表放于共享盘。
  4. 经理拾取图表，通过飞书通道回复给在外出差的人类用户。
