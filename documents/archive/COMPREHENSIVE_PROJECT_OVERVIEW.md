# OpenClaw Enterprise 完整系统功能复盘 (截止 2026-03-11)

经过对项目前后端和 Docker 构建脚本的全局解剖，当前这套系统已经不再是简单的外壳，而是一个自洽的、安全的 **微型企业虚拟运营沙箱引擎**。

它横跨了 `前端 React UI 管理层`、`Rust IPC 调度层`、`Docker 底层隔离网络` 三大部分。以下是目前系统已经 100% 跑通的核心功能全景图：

---

## 一、 系统与环境准备层 (The Foundation)

### 1. 纯净无头的环境母盘 (`openclaw_build`)
- **Docker 基座**: `openclaw-base:latest` 镜像。这不是普通的 Ubuntu，而是预装了所有极客需要的基础组件 (python3, pip, curl, git 等)，并在 `/workspace` 挂载就绪的**API执行级服务器**。
- **配置清洗与脱敏**: 在打包母盘前，有一个 Python 脚本专门清理了可能遗留在开发者本机的密钥与敏感日志，确保每个数字员工“出生”都是一张白纸。

### 2. 双驱网络基建 (Network Topology)
整个数字企业的通信生命线在首次打开 App（Rust 的 `setup` 钩子）时就已经悄悄打通：
- **`openclaw-intranet`**: 这是绝对隔离的内部局域网，不通外网（类似于断网的机房内网）。
- **`openclaw-internet`**: 这是对外通信网。
- **企业共享盘**: 强制在宿主机拉起 `/tmp/openclaw_enterprise_shared` 供各沙箱间进行物理文件高速交换。

---

## 二、 员工全生命周期管理层 (Employee Lifecycle)

### 1. SaaS体验的员工入职向导 (`Sidebar.tsx`)
- **三步引导 (Wizard)**：点击左栏加号，调出极具质感的多步弹窗：
  1. **Identity 定制**: 设置名字、角色、内存 (256m/512m/1g) 和 CPU 核心限制。
  2. **Providers 获取**: 自动从全局系统设置里透传大模型 API 的通行证。
  3. **Channels 赋权**: 提供 飞书、Lark、Discord 机器人的授权钥匙孔。
- **异步静默拉起 (`lib.rs` -> `add_employee`)**: 用户在界面点击“创建”时，底层会开启一个**专属线程**，默默替该员工复制一份属于它自己的工作区数据卷（Volume），整个过程行云流水，UI 完全不卡顿。

### 2. 黑魔法：动态配置注入 (Config Injection)
- **精准狙击脱敏文件**: 员工创建的瞬间，Rust 底层使用了**策略模式 (Strategy Pattern)**，强行潜入该员工专属的数据卷内，重写 `.openclaw/openclaw.json` 和 `models.json`。
- **环境级灌入**: 在员工开机（启动容器）时，Rust 会把员工的通道密码等敏感信息写死在 `.openclaw.env`，并作为 `--env` 加入该集装箱。这意味着代码**不需要经过宿主机硬盘和明文暴露**，员工的内生系统一觉醒来就已经获得了联网超能力。

---

## 三、 C 端大本营 (Dashboard & Resource Monitor)

目前的 `Dashboard.tsx` 已经是一块宏大且精细的总控看板：
- **API Provider 供应站**: 包含一键快速预填 OpenRouter/Anthropic/DeepSeek 后门。输入的密钥会经过遮罩脱敏。
- **物理资源实时看板**: 能计算当前有几个集装箱正在燃烧机器内存（例如：运行状态绿灯亮起，资源使用率滚动条填充）。
- **员工信息全景**: 支持在 Dashboard 里点击修改现有员工信息（名称、限制）。
- **基座重载**: 随时可以更改公司采购的新型号母盘（Base Image）或者换一套标准模板文件。

---

## 四、 内网监控与绝对控制 (Phase 5 协作黑盒)

这部分是**此项目和市面上绝大部分云端多 Agent 编排框架的核心护城河**：

### 1. 网络黑牢与外网开关 (Internet Isolation)
- 在 Dashboard 的 `Enterprise Intranet` 模块卡片上，针对每一个员工名字旁边都有一个“地球仪”开关（断网/全入网）。
- 点击断网，底层立刻执行 `docker network disconnect openclaw-internet [员工容器]`。
- **它的意义**：比如公司里需要一个处理税务机密数据的 AI 员工，只要把他网络一掐，他依然能通过内部网 (`intranet`) 接收指令并输出报表，但他永远没有能力将机密文件通过 `curl` 发送到互联网上的任何一台服务器。**做到了物理级别的防泄露。**

### 2. 跨沙箱超高速消息总线 (Local.Bus)
- 系统在左下角的菜单中内置了一个带有绿色呼吸灯的小型电台雷达图标。
- 点开即召唤出 **BusMonitor**。
- 这是一套基于 `企业共享挂载盘/.bus` 隐藏目录的消息队列。
- 当处于底层容器的员工们通过指令在内网互相呼叫时，**这个监控面板会毫秒级像谍战电台一样侦听到所有的指令走向并打卡时间戳。**

---

## 五、 交互工作台 (Workspace)

回到主界面：右侧工作台包含了三个主要切页：
1. **Chat & Terminal**: 真正的终端透传。在上面和员工聊天（通过他脑袋里的飞书等服务），在下面直接执行 `exec_sandbox` 敲击 Linux `ls`, `pip install` 命令。
2. **Files & Editor**: `FileTree.tsx` 接通了宿主机 OS 级的 `notify_rust` Watcher。如果沙箱内的员工偷偷创建了一个新文件，UI 树也会跟着瞬间产生一个分支。
3. **Gateways**: 单兵系统的网关随时重装。

目前 OpenClaw Enterprise 已经具备**完美的数字员工注册、物理隔离沙箱启停、双网关热切换拦截、配置穿透注入、以及最核心的高内聚总线通信体系**。这代表目前产品从 POC 走向了真正强健的企业级架构框架。
